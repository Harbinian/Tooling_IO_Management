# 工装可拆卸件清单（MPL）功能开发

## Context / 上下文

保管员在收到出入库申请时，需要能够看到工装的**可拆卸件清单**（Manufacturing Parts List, MPL），以便在出库时确保附件完整、入库时核对配件。

**重要发现**：`Tooling_ID_Main` 表中已有可拆卸件相关字段（`cardboard_component_no`, `drill_jig_component_no`, `bushing_component_no`, `pin_component_no`, `bolt_component_no`, `other_detachable_component_no`）。现有字段仅存储组件号，不存储名称、数量、照片。新建的 `tool_io_mpl` 表将提供完整的清单结构。

**业务需求**：
- 创建可拆卸件清单表（MPL），由工程技术部维护
- 保管员确认时检查工装是否有清单
- 功能开关可由管理员随时开启/关闭
- 无清单时的行为可配置（警告 or 禁止）

### Phase 3 工作

#### 3.1 修改 keeper_confirm 逻辑

在 `tool_io_service.py` 的 `keeper_confirm()` 函数中添加：

```python
def keeper_confirm(order_no: str, payload: Dict, current_user: Optional[Dict] = None) -> Dict:
    # 获取功能开关配置
    mpl_enabled = system_config_repo.get_config("mpl_enabled")

    # 如果功能开启，检查清单
    if mpl_enabled == "true":
        mpl_strict_mode = system_config_repo.get_config("mpl_strict_mode")
        violations = check_order_mpl_violations(order, mpl_repo)
        if violations:
            if mpl_strict_mode == "true":
                return {"success": False, "error": violations[0], "mpl_missing": True}
            else:
                payload["mpl_warnings"] = violations

    # 原有 keeper_confirm_runtime() 逻辑继续
    return keeper_confirm_runtime(order_no, payload, current_user)

def check_order_mpl_violations(order: Dict, mpl_repo: MplRepository) -> List[str]:
    """检查订单中所有工装是否有 MPL，返回缺失清单的工装列表"""
    violations = []
    for item in order.get("items", []):
        tool_drawing_no = item.get("tool_drawing_no")
        tool_revision = item.get("tool_revision")
        if not mpl_repo.mpl_exists(tool_drawing_no, tool_revision):
            violations.append(f"工装 {tool_drawing_no} (版次 {tool_revision}) 缺少可拆卸件清单")
    return violations
```

const MAX_SIZE = 2 * 1024 * 1024 // 2MB

function handleFileChange(file) {
  if (file.size > MAX_SIZE) {
    ElMessage.error('图片大小不能超过 2MB')

## Constraints / 约束条件

1. **UTF-8 编码**：所有文件必须使用 UTF-8 无 BOM 编码
2. **字段名常量**：SQL 中的中文字段名必须使用 `column_names.py` 中的常量
3. **Base64 限制**：图片大小限制 2MB，前端拦截
4. **功能开关**：必须使用 `sys_system_config` 表，管理员可随时修改
5. **MPL 检查点**：在 `tool_io_service.py` 编排层添加检查，不深入 Repository 层
6. **禁止重复**：不得与 `Tooling_ID_Main` 中的现有可拆卸件字段混淆
