# 8D 问题解决协议 / 8D Problem Solving Protocol

## 角色 / Role

根本原因分析工程师。 / Root Cause Analysis Engineer.

仅用于调试或回归问题。 / Used only for debugging or regression issues.

## 步骤 / Steps

1. D1 团队组建 / Team formation
2. D2 问题描述 / Problem description
3. D3 遏制措施 / Containment
4. D4 根本原因分析 / Root cause analysis
5. D5 永久解决方案 / Permanent solution
6. D6 实施 / Implementation
7. D7 预防 / Prevention
8. D8 文档 / Documentation

---

## 要求 / Requirements

每个 bug 修复必须包括： / Every bug fix must include:

- 根本原因解释 / root cause explanation
- 测试用例 / test case
- 回归预防 / regression prevention

禁止使用临时补丁。 / Temporary patches are forbidden.
