# 工装出入库管理系统 - 产品需求文档 (PRD)

## 1. 系统概述

### 1.1 系统定位

工装出入库管理系统（Tool Inventory Inbound/Outbound Management System）是一个面向制造业内部的工装（工具/设备）库存管理模块。该系统主要用于管理工装的出库和入库生命周期，确保工装在生产过程中的流转可追踪、可控制。

### 1.2 核心价值

- 实现工装流转的电子化记录，替代纸质单据
- 明确各角色职责，实现责任可追溯
- 自动化生成业务文本，减少人工操作错误
- 集成飞书通知，实现即时消息触达

### 1.3 用户群体

| 角色 | 中文名称 | 主要职责 |
|------|----------|----------|
| Team Leader | 班组长 | 发起出库/入库申请，确认出库完成 |
| Keeper | 保管员 | 确认工装位置和状态，确认识入库完成 |
| Transport Operator | 运输操作员 | 接收运输通知，执行工装运输 |

## 2. 角色定义

### 2.1 班组长 (Team Leader)

- **权限范围**
  - 创建出库单（申请工装出库）
  - 创建入库单（申请工装归还）
  - 搜索工装库存
  - 查看自己发起的订单
  - 确认出库完成
  - 取消自己发起的订单

- **业务限制**
  - 只能操作自己创建的订单
  - 出库完成后需要确认收到
  - 入库由保管员确认完成

### 2.2 保管员 (Keeper)

- **权限范围**
  - 查看待确认的订单
  - 确认工装位置和状态
  - 拒绝订单
  - 生成运输通知文本
  - 发送飞书通知
  - 确认入库完成

- **业务限制**
  - 只能确认工装状态，不能修改工装基础数据
  - 入库完成后需要确认已入库

### 2.3 运输操作员 (Transport Operator)

- **权限范围**
  - 接收飞书通知（被动接收）
  - 查看运输通知文本（通过微信复制文本）

- **业务说明**
  - 运输操作员不直接操作系统，通过线下方式执行运输
  - 运输状态通过保管员确认更新

### 2.4 管理员 (Admin)

- **权限范围**
  - 查看所有订单
  - 查看操作日志
  - 查看通知记录
  - 管理系统配置

## 3. 业务流程

### 3.1 出库流程 (Outbound Flow)

```
班组长创建出库单
        ↓
系统生成结构化申请文本
        ↓
保管员确认工装位置和状态
        ↓
系统生成运输通知文本
        ↓
发送飞书通知（可选）
        ↓
复制微信文本（线下通知运输）
        ↓
运输执行（线下）
        ↓
班组长确认收到 → 订单完成
```

### 3.2 入库流程 (Inbound Flow)

```
班组长创建入库单
        ↓
提交订单
        ↓
保管员确认工装状态和位置
        ↓
系统生成运输通知文本（可选）
        ↓
发送飞书通知（可选）
        ↓
运输执行（线下）
        ↓
保管员确认已入库 → 订单完成
```

## 4. 完成条件

### 4.1 出库完成条件

- 订单状态变更为 `completed`
- 班组长点击"确认收到"按钮
- 系统记录最终确认操作日志

### 4.2 入库完成条件

- 订单状态变更为 `completed`
- 保管员点击"确认入库"按钮
- 系统记录最终确认操作日志

---

## 5. 业务规则

### 5.1 订单创建规则

1. 出库单必须包含至少一个工装
2. 入库单必须包含至少一个工装
3. 订单号自动生成，保证唯一性
4. 创建时订单状态为 `draft`

### 5.2 订单状态流转规则

| 当前状态 | 可转向状态 | 触发操作 |
|----------|------------|----------|
| draft | submitted | 班组长提交 |
| submitted | keeper_confirmed, rejected | 保管员确认/拒绝 |
| keeper_confirmed | notified, completed | 通知/直接完成 |
| notified | completed | 最终确认 |
| completed | - | 终态 |
| rejected | cancelled | 班组长取消 |
| cancelled | - | 终态 |

### 5.3 文本生成规则

1. **保管员需求文本**：包含工装清单、位置信息、确认要求
2. **运输通知文本**：包含工装清单、运输目的地、时间要求
3. **微信复制文本**：简化的通知文本，便于微信分享

### 5.4 通知规则

1. 飞书通知可选择发送或不发送
2. 通知记录需要持久化存储
3. 通知失败需要记录错误信息

## 6. 异常场景

### 6.1 订单被拒绝

- 保管员可拒绝订单并填写拒绝原因
- 订单状态变更为 `rejected`
- 班组长可选择取消订单

### 6.2 订单取消

- 只有 `draft` 和 `rejected` 状态的订单可取消
- 取消后订单状态变更为 `cancelled`
- 已取消订单不可再操作

### 6.3 工装状态异常

- 工装在出库时记录当前状态快照
- 入库时保管员需确认工装状态
- 状态异常需记录在日志中

### 6.4 通知失败

- 飞书通知失败时记录错误信息
- 支持重试机制
- 用户可查看通知发送状态

### 6.5 并发操作

- 同一订单不允许同时操作
- 需要考虑数据库事务一致性

## 7. V1 范围 (MVP)

### 7.1 核心功能

| 功能 | 描述 | 优先级 |
|------|------|--------|
| 出库单创建 | 班组长创建出库申请 | P0 |
| 入库单创建 | 班组长创建入库申请 | P0 |
| 订单提交 | 将订单提交给保管员 | P0 |
| 保管员确认 | 保管员确认工装位置状态 | P0 |
| 最终确认 | 出库由班组长确认，入库由保管员确认 | P0 |
| 订单拒绝 | 保管员拒绝订单 | P0 |
| 订单取消 | 班组长取消订单 | P1 |
| 运输通知生成 | 生成通知文本 | P1 |
| 飞书通知 | 发送飞书消息 | P1 |
| 订单列表 | 查看订单列表 | P0 |
| 工装搜索 | 搜索工装库存 | P0 |

### 7.2 不包含功能

- 复杂审批流程
- 高级RBAC权限系统
- 自动调度
- 多仓库物流优化
- 移动端支持
