# 需求：首页门户导航

## 基本信息 / Basic Information
- 需求编号：REQ-20260409-001
- 创建日期：2026-04-09

## 5W2H 分析 / 5W2H Analysis

### What - 核心需求
**功能描述**：
在 `frontend/src/pages/` 下新增门户导航页面 `/home`，作为登录用户的主入口。页面以大图标 + 业务域名称的形式展示入口卡片，用户点击后跳转到对应业务域的默认页面。

**输入**：
- 当前登录用户的角色和权限信息（来自 session store）

**输出**：
- 根据用户权限动态渲染的门户入口卡片
- 无权限的业务域卡片不显示

### Why - 动机与价值
**业务背景**：
当前系统登录后直接跳转到 Dashboard，但 Dashboard 主要展示数据统计，对于新用户或不熟悉系统的用户来说，缺少一个清晰的业务入口导航。

**预期价值**：
- 新用户能快速了解系统功能模块
- 提供清晰的业务域跳转入口
- 根据权限动态显示，用户只看到有权限访问的模块

**成功标准**：
- 登录用户访问 `/home` 能看到所有有权限的业务域入口
- 无权限的业务域入口不显示
- 点击入口卡片能正确跳转到对应业务域

### When - 时间线
**期望完成时间**：

**使用频率**：

**优先级**：
- P1

### Where - 使用场景
**使用环境**：
- 生产环境浏览器访问

**集成需求**：
- 登录后默认跳转
- 路由 `/home` 需要登录权限

**约束条件**：
- 需与现有 MainLayout 侧边栏导航共存
- 入口卡片数量应适中，避免过多影响体验

### How - 实现方式
**建议实现方式**：
1. **页面位置**：`frontend/src/pages/home/HomePage.vue`
2. **URL 路由**：`/home`，登录后默认跳转
3. **门户卡片设计**：
   - 大图标 + 业务域名称
   - 卡片式布局，响应式网格
   - 悬停效果（轻微上浮 + 阴影加深）
4. **权限控制**：
   - 通过 session store 获取用户权限
   - 路由级别权限控制
   - 组件级别权限过滤

**技术栈**：
- Vue 3 Composition API
- Element Plus 组件
- CSS 变量（遵循现有规范）
