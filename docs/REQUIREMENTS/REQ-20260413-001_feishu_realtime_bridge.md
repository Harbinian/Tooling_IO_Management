## 基本信息 / Basic Information

- **创建日期**: 2026-04-13
- **状态**: 已完善
- **需求类型**: 功能开发 (00001-09999)

## 5W2H 分析 / 5W2H Analysis

### What - 核心需求

**功能描述**:
建立飞书与 Claude CLI 之间的长连接双向实时通信服务。飞书发送指令到服务，服务调用 Claude CLI 执行，执行结果通过飞书 bot 主动推送回飞书。

**技术特性**:
- 长连接模式（WebSocket），飞书与服务的连接保持活跃
- 主动推送模式，服务可主动向飞书发送消息
- 常驻后台运行，作为守护进程
- 跨项目复用，不绑定特定业务逻辑

**输入**:
- 飞书消息（通过长连接接收）
- Claude CLI 执行结果（通过 stdout 捕获）

**输出**:
- 飞书消息（通过 bot 主动推送）
- CLI 执行日志

### Why - 动机与价值

**业务背景**:
飞书和 Claude CLI 之间的实时联动需求明确，但现有实现（如 feishu-command-bridge）采用轮询模式，效率低且架构不清晰。需要一个真正的长连接方案。

**核心价值**:
1. **实时性**: 长连接替代轮询，消息秒级响应
2. **可复用性**: 独立服务设计，跨项目部署
3. **统一性**: 替代所有现有的桥接功能

**成功标准**:
- 飞书发送指令 → 服务接收 → Claude CLI 执行 → 结果推回飞书，全流程 < 5 秒
- 服务可脱离主项目独立运行
- 其他项目可通过简单配置复用此服务

### When - 时间线

**优先级**: P0（紧急）

**期望完成时间**: 尽快

**使用频率**: 常驻运行

### Where - 使用场景

**使用环境**:
- 开发者本地机器（开发验证）
- 远程服务器（生产部署）

**技术约束**:
- 飞书开放平台长连接支持（需确认企业已开通相关权限）
- Claude CLI 已安装且可远程调用
- 网络可达性（服务与飞书服务器之间）

### How - 实现方式

**建议实现方式**:
1. **独立 Python 服务**: 使用 FastAPI/Flask-SocketIO 构建
2. **飞书 SDK**: 使用飞书官方 SDK（lark-oapi）的 WebSocket 模式
3. **CLI 调用**: subprocess 包装 Claude CLI 调用
4. **消息推送**: 飞书 bot API 主动推送

**参考系统**:
- 飞书官方 lark-oapi SDK（长连接模式）
- 现有 `scripts/feishu_bridge/`（了解消息格式）

**技术栈**:
- Python 3.10+
- lark-oapi（飞书 SDK）
- FastAPI / Flask-SocketIO（可选）
- subprocess（CLI 调用）
