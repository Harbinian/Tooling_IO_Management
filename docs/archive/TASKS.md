# 开发任务分解

## 概述

本文档将开发工作分解为多个阶段，明确每个阶段的目标、输入、输出和负责agent。

---

## 阶段总览

| 阶段 | 名称 | 负责agent | 类型 |
|------|------|-----------|------|
| Phase 1 | 数据库脚本审计与字段修复 | Codex | 修复 |
| Phase 2 | 服务层包装与API对齐 | Codex | 扩展 |
| Phase 3 | 保管员工作流完善 | Codex | 扩展 |
| Phase 4 | 通知持久化与飞书集成 | Codex | 扩展 |
| Phase 5 | 最终确认与审计完成 | Codex | 扩展 |
| Phase 6 | 前端集成 | Gemini | 新增 |

---

## Phase 1: 数据库脚本审计与字段修复

### 目标

审计现有数据库实现，修复 schema 不一致问题。

### 输入

- database.py
- docs/DB_SCHEMA.md

### 输出

- 修复缺失字段的 ALTER TABLE 语句
- 更新的表创建脚本

### 任务列表

| 任务 | 类型 | 说明 | 负责 |
|------|------|------|------|
| 1.1 | 修复 | 主表添加"工装数量"字段 | Codex |
| 1.2 | 修复 | 主表添加"已确认数量"字段 | Codex |
| 1.3 | 修复 | 主表添加"最终确认人"字段 | Codex |
| 1.4 | 修复 | 明细表添加"确认时间"字段 | Codex |
| 1.5 | 修复 | 明细表添加"出入库完成时间"字段 | Codex |
| 1.6 | 审查 | 验证所有 CRUD 函数与 schema 一致性 | Codex |

### 完成标准

- 所有缺失字段已添加
- CRUD 函数不再报错

---

### 输入

- database.py（现有函数）

- web_server.py

### 输出

- Flask API 路由
- 服务层封装

| 2.1 | 扩展 | Flask 框架搭建/完善 | Codex |
| 2.2 | 扩展 | 订单创建 API | Codex |
| 2.3 | 扩展 | 订单列表查询 API | Codex |
| 2.4 | 扩展 | 订单详情查询 API | Codex |
| 2.5 | 扩展 | 订单提交 API | Codex |
| 2.6 | 扩展 | 工装搜索 API | Codex |
| 2.7 | 扩展 | 参数验证中间件 | Codex |
| 2.8 | 扩展 | 获取操作日志 API | Codex |
| 2.9 | 扩展 | 获取待确认订单 API | Codex |

### 完成标准

- API 可访问
- 复用现有 database.py 函数

## 任务类型说明

| 类型 | 说明 |
|------|------|
| 修复 | 修复现有实现中的问题 |
| 扩展 | 在现有基础上扩展功能 |
| 新增 | 全新添加的功能 |

## 任务依赖关系

```

    ↓

    ↓
Phase 3 (保管员工作流) ← Phase 2
    ↓
Phase 4 (通知) ← Phase 3
    ↓
Phase 5 (最终确认) ← Phase 4
    ↓
Phase 6 (前端) ← Phase 5
```
