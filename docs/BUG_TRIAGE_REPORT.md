## 观察到的症状 / Observed Symptom

`backend/services/order_query_service.py:115` 中的 `search_tool_inventory` 函数：

```python
def search_tool_inventory(filters: Dict) -> Dict:
    """Search tool inventory."""
    return search_tools(filters)  # ❌ 传递字典，但 search_tools() 期望关键字参数
```

调用 `database.py` 中的 `search_tools`：

```python
def search_tools(
    keyword: str = "",
    status: Optional[str] = None,
    location: Optional[str] = None,
    page: int = 1,
    page_size: int = 20
) -> dict:
```

**问题**: `filters` 字典会被当作 `keyword` 字符串参数传入，导致运行时类型错误。

---

## 可能的根本原因 / Possible Root Cause

**直接原因**: `order_query_service.py` 的 `search_tool_inventory` 函数错误地假设 `database.search_tools()` 接受字典参数。

**调用链分析**:
```
order_query_service.search_tool_inventory(filters)
  ↓ 调用
database.search_tools(filters)  # ❌ 错误：filters 是 dict，不是 str
```

**正确调用应该是**:
```python
search_tools(
    keyword=filters.get("keyword"),
    status=filters.get("status"),
    location=filters.get("location"),
    page=filters.get("page_no", 1),
    page_size=filters.get("page_size", 20),
)
```

或使用 `**filters` 解包：
```python
search_tools(**filters)
```

### 观察到的症状

```
GET /tool-io-orders/TO-OUT-20260324-005/issues → 404 NOT FOUND
```

### 根本原因

前端调用 `/issues`，后端路由是 `/transport-issues` 和 `/report-transport-issue`。

| 操作 | 前端调用 | 后端路由 |
|------|---------|---------|
| 获取运输异常 | `/tool-io-orders/{order_no}/issues` | `/tool-io-orders/{order_no}/transport-issues` |
| 上报运输异常 | `/tool-io-orders/{order_no}/issues` | `/tool-io-orders/{order_no}/report-transport-issue` |

### 修复范围

修改 `frontend/src/api/orders.js` 中的API路径以匹配后端路由。
