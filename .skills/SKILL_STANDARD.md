# 技能文件编写标准 / Skill File Standard

---

## 必须包含的结构

1. **Frontmatter**（必填7字段）
   - `name` — kebab-case，与文件名一致
   - `executor` — Claude Code | Codex | Human
   - `auto_invoke` — 是否可被链式调用
   - `depends_on` — 上游技能列表
   - `triggers` — 触发命令列表
   - `rules_ref` — 引用的规则文件
   - `version` — 版本号

2. **目的 / Purpose**（≤ 3 句话，说明技能解决什么问题）
3. **上下文约束 / Context Constraints**（权限边界、禁止操作、依赖环境）
4. **与相邻技能的边界**（存在职责相近技能时必填，说明分工）
5. **执行步骤**（编号，每步有明确输出物）
6. **完成标准**（可客观验证的交付物清单）

---

## 体积限制

| 状态 | 阈值 | 处理方式 |
|------|------|---------|
| 正常 | ≤ 15KB | 无需处理 |
| 警告 | 15KB ~ 20KB | 须在 PR 说明中解释原因 |
| 阻断 | > 20KB | 必须外部化数据后方可合并 |

**外部化规则**：
- 路径/URL 列表 → `docs/` 目录 yaml 文件
- 角色权限矩阵 → `docs/rbac_matrix.yaml`
- 测试数据 → `tests/fixtures/`
- 重复流程片段 → 共享 include 文件

---

## 禁止项

- 禁止硬编码路径（使用配置文件引用）
- 禁止内联大量测试数据
- 禁止隐式依赖（所有依赖必须在 `depends_on` 中声明）
- 禁止与现有技能职责重叠超过 40% 而不声明边界
- 禁止在元技能中设置 `auto_invoke: true`

---

## Executor 判断标准

| 判断条件 | Executor 值 |
|---------|------------|
| 需要读写文件系统、执行终端命令 | `Claude Code` |
| 需要大规模代码生成或重构 | `Codex` |
| 需要人类判断业务语义或做决策 | `Human` |
| 仅生成文档或分析报告 | `Claude Code` |

---

## 双语要求

- 所有区块标题使用中英双语：`## 目的 / Purpose`
- 正文可选择主语言，但关键约束句必须双语
- Frontmatter 字段名使用英文

---

## 元技能特殊规定

元技能（即管理、审查或生成其他技能的技能）须遵守：
- 不得将自身列入审查范围（禁止循环依赖）
- `auto_invoke` 必须为 `false`，只允许手动触发
- 必须在 `## 上下文约束` 中声明"本技能为元技能"
