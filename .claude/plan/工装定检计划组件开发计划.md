# 工装定检计划组件开发计划

## Context

工装出入库管理系统需要新增**工装定检计划**组件，用于管理工装的定期检验计划与执行。需求背景：
- 当前系统只有 tool_io 出入库模块，工装维护保养功能缺失
- 物资保障部计划员需要发布和维护定检计划
- 运维安环部班组长需要执行定检（涉及 tool_io 出入库）
- 质量管理部检验员需要验收测量结果
- 测量报告需上传附件（Base64，2MB 限制）
- 保管员负责关闭任务，系统自动更新工装定检状态

**架构决策**：共库、统一认证、统一通知、独立状态机

---

## 数据模型

### 4 张新表（共库）

| 表名 | 用途 | 主键格式 |
|------|------|---------|
| `tool_io_inspection_plan` | 定检计划 | `DJP-{YYYYMMDD}-{SEQ}` |
| `tool_io_inspection_task` | 定检任务 | `DJT-{YYYYMMDD}-{SEQ}` |
| `tool_io_inspection_report` | 测量报告 | `RPT-{YYYYMMDD}-{SEQ}` |
| `tool_io_tool_inspection_status` | 工装定检状态 | `serial_no`（关联外部表） |

**测量报告附件**：`attachment_data` 字段使用 `NVARCHAR(MAX)` 存储 Base64，2MB 限制，参考 MPL 设计

---

## 状态机

### 任务状态流转

```
[pending] → [received] → [outbound_created] → [outbound_completed]
    → [in_progress] → [report_submitted] → [accepted]
    → [inbound_created] → [inbound_completed] → [closed]
```

### 与 tool_io 集成点

- 任务 `received` → 班组长创建出库单 → `outbound_created`
- 出库单 `completed` → 自动推进任务至 `outbound_completed`
- 任务 `accepted` → 班组长创建入库单 → `inbound_created`
- 入库单 `completed` → 自动推进任务至 `inbound_completed`
- 任务 `inbound_completed` → 保管员关闭 → `closed` → **自动更新工装定检状态表**

## 核心设计决策

1. **附件存储**：复用 MPL 的 Base64 方案，`NVARCHAR(MAX)` 字段，前端 `FileReader.readAsDataURL()`，2MB 限制
2. **状态同步**：tool_io 订单完成不直接回调，而是通过**轮询**或**入库后主动调用**方式推进定检任务状态
3. **计划→任务自动生成**：计划发布时，根据 `tools` 列表自动批量创建任务
4. **工装定检状态表**：保管员关闭任务时写入/更新，支持后续查询工装定检到期状态
